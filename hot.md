## Key Takeaways
- Translator Joe Sachs's introduction is itself a load-bearing source here: it argues the standard English translation tradition mistranslates three central terms — *hexis* ("habit" → should be "active condition"), *energeia* ("activity" → should be "being-at-work"), and *to kalon* ("noble"/"fine" → should be "the beautiful") — and several pages in this ingest carry that correction explicitly rather than using the more familiar English terms.
- On akolasia specifically: "dissipation" is Sachs's deliberate, defended choice, but the user's instinct toward "indulgence" is not a misreading — it's W. D. Ross's standard alternative translation for the same word. The one thing "indulgence" risks losing on its own is the settled, unregretful, chosen-as-a-principle character of the vice (which is what actually separates it from akrasia) — worth mentally pairing it with "as a character trait, not a momentary lapse."
- **The core diagram-selection principle for this vault**, arrived at the hard way: every diagram grammar carries an implicit claim about the kind of relationship it depicts — arrows imply flow/causation/precedence, radiating branches imply generation, numeric axes imply measurement, `<|--` implies static is-a. Only use a diagram type when that implicit claim is *literally true* of the content, not just evocative. Never invent numbers to populate a chart (radar/xyChart scores with no textual basis are a provenance violation, not a stylistic choice). Where content is a genuine hierarchy, classification, or proportion, diagram it directly. Where content is an argument chain, an elimination-by-exclusion argument, or independent parallel facets, there is often *no* honest diagram for it — prose is the correct answer, not a consolation prize. "Diagram every page" was the wrong default; "diagram only what has a diagrammable shape" is the right one.
- Diagram-type lesson from the household-justice case: Venn diagrams show *partial* overlap between sets, and fail when (a) the actual relationship is strictly cumulative/nested (that's inheritance, not overlap), or (b) some items being compared belong to *none* of the depicted sets, since Venn has no clean way to label "outside all circles."
- Mermaid `classDiagram` rendering lesson: avoid `note for ClassName "..."` — it renders as a floating box disconnected from its subject. Put descriptive text directly in the class body instead (plain lines with no `+`/`-`/`#` prefix render fine as free text).
- Workflow-tool lesson: passing a JSON array via the `args` parameter is not reliable inside the sandboxed script — it arrived as a stringified value despite being sent as a real array. Fix: bake a known, fixed list as a literal directly into the script body instead of threading it through `args`.
- The `/mermaid` skill routes to ~30 sub-skill reference files at `~/.agents/skills/mermaid/references/<type>.md` and explicitly warns not to guess syntax from pretraining. When delegating to sub-agents that may lack Skill-tool access, have them `Read` the specific reference file directly by path instead.
- Greek-source lesson: Perseus's Scaife Viewer, the classic Hopper interface, Perseus-under-PhiloLogic, and Logeion all render their actual text via JavaScript, so WebFetch only ever sees the app shell, never the Greek. The reliable path for this vault is PerseusDL's own GitHub repo (`github.com/PerseusDL/canonical-greekLit`, e.g. `data/tlg0086/tlg010/tlg0086.tlg010.perseus-grc2.xml` for the *Nicomachean Ethics*) — `curl` the raw TEI XML into the scratchpad and `grep -oP ".{N}TERM.{N}"` for bounded-context extraction (the file is one giant line per paragraph, so plain `grep -n` alone returns unreadable walls of text). Bekker page/line milestones are literal `<milestone unit="page/line" resp="Bekker" n="..."/>` tags — check the milestone immediately before a match's line number to get the citation, don't trust memory for exact line numbers even on well-known passages.

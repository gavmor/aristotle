## Key Takeaways
- Translator Joe Sachs's introduction is itself a load-bearing source here: it argues the standard English translation tradition mistranslates three central terms — *hexis* ("habit" → should be "active condition"), *energeia* ("activity" → should be "being-at-work"), and *to kalon* ("noble"/"fine" → should be "the beautiful") — and several pages in this ingest carry that correction explicitly rather than using the more familiar English terms.
- On akolasia specifically: "dissipation" is Sachs's deliberate, defended choice, but the user's instinct toward "indulgence" is not a misreading — it's W. D. Ross's standard alternative translation for the same word. The one thing "indulgence" risks losing on its own is the settled, unregretful, chosen-as-a-principle character of the vice (which is what actually separates it from akrasia) — worth mentally pairing it with "as a character trait, not a momentary lapse."
- Diagram-type lesson from the household-justice case: Venn diagrams show *partial* overlap between sets, and fail when (a) the actual relationship is strictly cumulative/nested (each item has everything the previous item has, plus more — that's inheritance, not overlap), or (b) some items being compared belong to *none* of the depicted sets, since Venn has no clean way to label "outside all circles" — that information gets stranded in a caption, defeating the point of a diagram. Check for both conditions before reaching for Venn again; a treemap, classDiagram, or plain table is usually the better fit for one-dimensional or cumulative data.
- Mermaid `classDiagram` rendering lesson: avoid `note for ClassName "..."` for anything that matters — Mermaid renders these as separate floating boxes wherever the layout engine decides to put them, disconnected from the class they annotate. Put descriptive text directly in the class body instead (plain lines with no `+`/`-`/`#` prefix render fine as free text inside the box) — this is what actually keeps an annotation visually anchored to its subject.
- Workflow-tool lesson: passing a JSON array via the `args` parameter is not reliable inside the sandboxed script — it arrived as a stringified value (`files.map is not a function`) despite being sent as a real array in the tool call. Fix: for a known, fixed list of items, just bake the array as a literal directly into the script body rather than threading it through `args`.
- The `/mermaid` skill is a router to ~30 sub-skill reference files at `~/.claude/skills/mermaid/references/<type>.md`, one per diagram type — it explicitly warns not to guess syntax from pretraining since it varies significantly by type. When delegating diagram work to sub-agents that may not have Skill-tool access, have them `Read` the specific reference file directly by path instead of invoking the skill — this worked reliably across all 15 agents in this pass.
